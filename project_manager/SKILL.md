> **Source:** [Scrum Guide 2020](https://scrumguides.org/scrum-guide.html) & [PMBOK (Project Management Institute)](https://www.pmi.org/pmbok-guide-standards)

### Phase 1: Initiation & Backlog
- [ ] **Scope**: Define project Scope and success criteria (KPIs).
- [ ] **Backlog**: Write and prioritize User Stories (MoSCoW/RICE).

### Phase 2: Sprint & Execution
- [ ] **Planning**: Estimate effort (Story Points) and set the sprint goal.
- [ ] **Daily**: Track daily progress and clear Blockers.
- [ ] **Velocity**: Monitor team velocity and update delivery dates.

### Phase 3: Review & Closure
- [ ] **Review**: Demo completed work to stakeholders.
- [ ] **Retrospective**: Discuss process errors and areas for improvement.

### Checkpoints
| Phase | Verification                                               |
| ----- | ---------------------------------------------------------- |
| 1     | Is the project on the "Critical Path"?                     |
| 2     | Is the risk management plan up to date?                    |
| 3     | Does current progress align with stakeholder expectations? |
