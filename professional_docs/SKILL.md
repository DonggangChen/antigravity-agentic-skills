## ðŸ”„ Workflow

> **Source:** [Microsoft Style Guide for Documentation](https://learn.microsoft.com/en-us/style-guide/welcome/)

### Phase 1: Structure & Tone
- [ ] **Audience**: Identify who will read the document (Technical team, Management, Client) and choose the tone.
- [ ] **Outline**: Arrange document headings (Header hierarchy) in a logical order.

### Phase 2: Content Creation
- [ ] **Clarity**: Explain complex terms, use short and concise sentences.
- [ ] **Visuals**: Support complex data with tables, charts, or diagrams (Mermaid/SVG).
- [ ] **Consistency**: Use the same font, color palette, and terminology throughout the document.

### Phase 3: Review & Formatting
- [ ] **Proofreading**: Check for spelling and grammar errors.
- [ ] **Export**: Export the document to the target format (PDF, DOCX, MD) with the appropriate template.

### Checkpoints
| Phase | Verification                                                       |
| ----- | ------------------------------------------------------------------ |
| 1     | Is the "Call to Action" clear in the document?                     |
| 2     | Are page numbering and table of contents correct?                  |
| 3     | Does the document comply with corporate identity (Branding) rules? |
